dns-edge GoEdge Provider 接入方案

版本：v3.0  |  日期：2026-06-25  |  分支：feature/goedge-provider

1. 背景与目标

1.1 GoEdge 的 DNS 体系

GoEdge 是一套自托管 CDN 管理平台，同时支持 EdgeDNS 角色与 EdgeNode 平级，专门负责边缘 DNS 解析：

EdgeAdmin（Web 管理后台）
    │ gRPC / REST
    ▼
EdgeAPI（中心管理服务，有自己的 MySQL）
    ├─ DNS Provider 接口 ──► 外部 DNS 后端（阿里云 / Cloudflare / dns-edge …）
    │
    ▼
EdgeDNS（边缘 DNS 节点，和 EdgeNode 地位相同）

1.2 dns-edge 的角色

dns-edge 既可以作为 GoEdge 的 DNS 提供商插件（customHTTP 模式），也可以作为原生 EdgeDNS 节点（edgeDNSAPI 模式）被 GoEdge 管理。两套接口可并存，差别在于 edgeDNSAPI 支持更丰富的地理路由线路配置（省份/ISP/国家），实现智能 DNS 分流。

1.3 目标

  1. 已完成 实现 GoEdge customHTTP Provider 接口（POST /goedge/dns）
  2. ✅ 已完成 实现 GoEdge edgeDNSAPI 兼容接口（POST /NS*Service/*），支持智能 DNS 地理分流配置
  3. ✅ 已完成 在公司 edgeapi repo 里实现 NS 商业版服务端，让 GoEdge 管理后台具备 EdgeDNS 集群管理能力

2. 架构说明

2.1 存储与同步架构

中心存储只有 edgeapi MySQL（GoEdge 本身就依赖 MySQL，这是固有要求）。dns-edge 节点不直连任何数据库，记录全部存内存，通过两种方式从 edgeapi 同步：

GoEdge EdgeAdmin
    │
    ▼
GoEdge EdgeAPI（edgeapi，MySQL）
    │
    ├─ 推送（低延迟，变更即通知）
    │    edgeapi → dns-edge /internal/sync
    │
    └─ 轮询（兜底 + 节点扩容）
         dns-edge 定期拉取 /NSRecordService/ListNSRecords
         │
         ▼
┌─────────────────────────────────────────────┐
│  dns-edge 节点（每个边缘节点）                │
│                                             │
│  ZoneStore（内存） ◄── 推送 / 轮询同步        │
│       │                                     │
│  [DNS Handler]                              │
│  [内部 REST API] /api/v1/...                │
└─────────────────────────────────────────────┘

5. 已完成：edgeDNSAPI 服务端（Phase 14）

Phase 14 — ✅ 已完成

5.1 概述

dns-edge 实现这套接口后，GoEdge 可以把 dns-edge 当作原生 EdgeDNS 节点来管理，并通过 NSRouteService 配置地理路由线路，实现智能 DNS 分流。

接口格式：POST /<ServiceName>/<MethodName>，响应格式统一：

{ "code": 200, "message": "", "data": { ... } }

5.2 鉴权流程

POST /APIAccessTokenService/getAPIAccessToken
请求：{ "type": "user", "accessKeyId": "xxx", "accessKey": "xxx" }
响应：{ "code": 200, "data": { "token": "...", "expiresAt": 1234567890 } }

GoEdge 在 Token 过期前 600 秒自动刷新，后续所有请求携带 X-Edge-Access-Token header。

5.3 NSDomainService

路径请求参数响应
/NSDomainService/ListNSDomainsoffset int, size int{ nsDomains: [{id,name,isOn,isDeleted}] }
/NSDomainService/FindNSDomainWithNamename string{ nsDomain: {id,name} }

5.4 NSRecordService

路径请求参数说明
/NSRecordService/ListNSRecordsnsDomainId, offset, size分页列出记录，每条含 nsRoutes
/NSRecordService/FindNSRecordWithNameAndTypensDomainId, name, type查单条记录
/NSRecordService/FindNSRecordsWithNameAndTypensDomainId, name, type查多条记录
/NSRecordService/CreateNSRecordnsDomainId, name, type, value, ttl, nsRouteCodes []stringnsRouteCodes 写入 route_tags
/NSRecordService/UpdateNSRecordnsRecordId, name, type, value, ttl, nsRouteCodes, isOn更新记录
/NSRecordService/DeleteNSRecordnsRecordId软删除

5.5 NSRouteService（地理路由线路）

路径说明返回示例
/NSRouteService/FindAllDefaultWorldRegionRoutes世界区域[{name:"中国",code:"country:中国"}]
/NSRouteService/FindAllDefaultChinaProvinceRoutes中国省份[{name:"上海",code:"province:上海"}]
/NSRouteService/FindAllDefaultISPRoutes运营商[{name:"电信",code:"isp:电信"}]
/NSRouteService/FindAllAgentNSRoutes代理线路（暂不支持）[]
/NSRouteService/FindAllNSRoutes自定义线路（暂不支持）[]

5.6 nsRouteCodes ↔ route_tags 映射

nsRouteCodes（GoEdge 传入）route_tags（PG 存储）
[] 或 ["default"]""（默认路由）
["province:上海"]"province=上海"
["isp:电信"]"isp=电信"
["country:中国"]"country=中国"
["province:上海","isp:电信"]"province=上海;isp=电信"

转换规则：code 格式为 <维度>:<值>，存储时转为 <维度>=<值>，多个用 ; 分隔。

GoEdge EdgeAdmin
    │ 管理 NS 集群和节点
    ▼
GoEdge EdgeAPI（edgeapi，Phase 15 补全，MySQL 存储）

dns-edge（Phase 14 补全 edgeDNSAPI 服务端）
